# Vercel 部署指南

## 架构说明

本项目采用前后端分离架构，部署需要两个独立的服务：

| 组件 | 部署平台 | 说明 |
|------|---------|------|
| 前端 (React) | **Vercel** | 静态网站托管，自动构建部署 |
| 后端 (FastAPI) | **Railway / Render** | Python服务托管，需要持续运行 |

---

## 一、后端部署（Railway 推荐）

### 1.1 准备工作

确保 `backend/requirements.txt` 包含所有依赖：

```
fastapi>=0.104.0
uvicorn>=0.24.0
langgraph>=0.2.0
langchain>=0.3.0
langchain-openai>=0.2.0
pydantic>=2.0.0
python-dotenv>=1.0.0
httpx>=0.27.0
```

### 1.2 Railway 部署步骤

1. 访问 [railway.app](https://railway.app) 注册/登录
2. 点击 "New Project" -> "Deploy from GitHub repo"
3. 选择你的 GitHub 仓库
4. 配置以下内容：
   - **Root Directory**: `backend`
   - **Build Command**: `pip install -r requirements.txt`
   - **Start Command**: `uvicorn app:app --host 0.0.0.0 --port $PORT`
5. 在 Variables 中添加环境变量：

| 变量名 | 值 | 说明 |
|--------|-----|------|
| `KIMI_API_KEY` | `sk-xxx` | 你的 Kimi API Key |
| `KIMI_API_BASE` | `https://api.moonshot.cn/v1` | Kimi API 地址 |
| `KIMI_MODEL` | `moonshot-v1-8k` | 模型名称 |
| `SERVER_PORT` | `8000` | Railway 会自动分配 PORT |

6. 部署完成后，Railway 会生成一个域名如 `xxx.railway.app`
7. **记录这个域名**，后续前端需要配置

### 1.3 Render 部署步骤（备选）

1. 访问 [render.com](https://render.com) 注册/登录
2. 点击 "New" -> "Web Service"
3. 连接 GitHub 仓库
4. 配置：
   - **Root Directory**: `backend`
   - **Build Command**: `pip install -r requirements.txt`
   - **Start Command**: `uvicorn app:app --host 0.0.0.0 --port $PORT`
5. 添加环境变量（同上）
6. 部署完成后获得域名如 `xxx.onrender.com`

## 二、前端部署（Vercel）

### 2.1 准备工作

1. 将项目推送到 GitHub 仓库
2. 确保项目根目录有 `vercel.json`（已创建）

### 2.2 Vercel 部署步骤

1. 访问 [vercel.com](https://vercel.com) 注册/登录
2. 点击 "Add New" -> "Project"
3. 导入你的 GitHub 仓库
4. Vercel 会自动检测到 `vercel.json` 配置
5. 在 Environment Variables 中添加：

| 变量名 | 值 | 说明 |
|--------|-----|------|
| `REACT_APP_API_BASE` | `https://xxx.railway.app` | 替换为你的后端实际域名 |

6. 点击 "Deploy"
7. 部署完成后 Vercel 会生成域名如 `xxx.vercel.app`

### 2.3 自定义域名（可选）

1. 在 Vercel 项目设置 -> Domains 中添加自定义域名
2. 按提示配置 DNS 记录

## 三、部署前检查清单

### 3.1 代码检查

- [ ] 所有路径已改为相对路径（无 `e:\demo\chatbot\...` 等硬编码）
- [ ] `backend/.env` 中的 `KIMI_API_KEY` 已配置真实密钥
- [ ] `frontend/.env` 中的 `REACT_APP_API_BASE` 指向后端实际地址
- [ ] `.gitignore` 已排除 `.env` 文件（防止密钥泄露）

### 3.2 安全注意事项

| 注意事项 | 说明 |
|----------|------|
| **不要提交 .env 文件** | `.gitignore` 已配置排除，确保 `.env` 不在 Git 中 |
| **API Key 通过平台环境变量注入** | Railway/Render 和 Vercel 都支持环境变量管理 |
| **CORS 配置** | `app.py` 中已配置 `allow_origins=["*"]`，生产环境建议限制为前端域名 |
| **限流保护** | 已配置单会话每分钟10次调用限制 |

### 3.3 CORS 生产环境优化

部署后建议修改 `backend/app.py` 中的 CORS 配置：

```python
app.add_middleware(
    CORSMiddleware,
    allow_origins=["https://your-frontend.vercel.app"],  # 限制为前端域名
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)
```

## 五、常见问题

### Q1: 前端显示"网络连接失败"

**原因**：前端无法连接到后端 API

**解决**：
1. 检查 Vercel 环境变量 `REACT_APP_API_BASE` 是否正确
2. 检查后端是否正常运行
3. 检查浏览器控制台是否有 CORS 错误

### Q2: 后端部署后请求超时

**原因**：Railway/Render 免费套餐有冷启动延迟

**解决**：
1. 首次请求可能需要 30-60 秒冷启动
2. 升级到付费套餐可避免冷启动
3. 或使用 UptimeRobot 等服务定期 ping 保持活跃

### Q3: API Key 泄露风险

**解决**：
1. 确保 `.env` 文件在 `.gitignore` 中
2. 使用平台环境变量而非文件配置
3. 定期轮换 API Key

### Q4: 如何更新部署

**Vercel（前端）**：
- 推送代码到 GitHub 主分支，Vercel 自动重新部署
- 或在 Vercel Dashboard 手动触发 Redeploy

**Railway（后端）**：
- 推送代码到 GitHub 主分支，Railway 自动重新部署
- 注意：后端重新部署会导致短暂不可用
